Download Tutorial Sources

Before we start setting up the environment, let’s clone the tutorial sources and set the TUTORIAL_HOME environment variable to point to the root directory of the tutorial:

git clone https://github.com/redhat-developer-demos/kubernetes-tutorial &&\
export TUTORIAL_HOME="$(pwd)/kubernetes-tutorial"
cd $TUTORIAL_HOME

This tutorial was developed and tested with:

Start Minikube

There are series of commands as steps that allows you get started with minikube. The following section shows how to start minikube with minimal and required configurations:

minikube --profile devnation config set cpus 2 (1)
minikube --profile devnation config set memory 6144 (2)
minikube --profile devnation config set disk-size 50g (3)
minikube --profile devnation config set vm-driver virtualbox (4)
minikube --profile devnation config set kubernetes-version v1.17.0 (5)

1 Using 2 cpus which is good enough for the basics There is nothing in this series of exercises that is CPU intensive
2 Memory matters, especially when running traditional Java apps (e.g. Weblogic, Spring, JBoss)
3 More disk space for storing container images
4 Using virtualbox because it is available on all platforms. There a number of hypervisor options that you can choose with minikube
5 Setting the Kubernetes version explicitly so you know what you are getting 
minikube start --profile devnation (1)

1 Profiles allow you to manage different configurations
